# The Loop Wiki

This repo publishes [wiki.theloop.community](https://wiki.theloop.community),
the public wiki for The Loop's DJ course. Its `gh-pages` branch is what
GitHub Pages serves; `main` holds the site config and theme.

Content is filtered and pushed here from a shared Obsidian vault by tooling
in a separate repo, [`theloopwiki-tools`](https://github.com/djsprenk/theloopwiki-tools).

Built with [Quartz](https://quartz.jzhao.xyz/), a static site generator for
publishing digital gardens and notes.

## Local development

To preview the site with content already filtered into `../content` (see
`theloopwiki-tools`'s `publish.sh` / `filter.py`), run from this directory:

```
npx quartz build --serve --directory ../content
```

This builds and serves the site locally with live-reload, without deploying
anything. Use it to iterate on theme/config changes in `quartz.config.yaml`
and `quartz/styles/`.

## Locally-forked plugins

Some plugins are forked locally instead of pulled from
`github:quartz-community/*`, because their config options don't expose
everything we need (e.g. the footer plugin only exposes a `links` option,
not the message text itself). Forks live in `local-plugins/` at the repo
root — unlike `.quartz/plugins/`, this directory is tracked in git and
symlinked into `.quartz/plugins/` via:

```
npx quartz plugin add ./local-plugins/<name>
```

`quartz.config.yaml` then points that plugin's `source:` at the local path
instead of the GitHub repo.

**After editing a local plugin's `src/`, you must rebuild it manually**:

```
cd local-plugins/<name>
npx tsup
```

`npx quartz plugin update <name>` / `install --latest` do **not** pick up
source edits — they reported "using pre-built dist/" and left the stale
`dist/` in place even after `src/` changed. `npx tsup` is the only thing
confirmed to actually recompile the fork.

Current forks:

- `footer` — customizes the footer message to "The Loop © {year} - Created
  with Quartz v{version}", with "The Loop" linking to
  https://theloop.community. The links list (`GitHub`/`Discord Community`)
  was removed via `options.links: {}` in `quartz.config.yaml`.
- `folder-page` — adds a `hideListingWhenIndexed` option (default `true`) to
  `FolderContent`. Folders with a hand-authored index (either a real
  `index.md`, or the Obsidian "folder note" convention of a file named the
  same as its parent folder, e.g. `The Loop Academy/The Loop Academy.md`)
  show only that written content — the auto-generated by-date file listing
  is skipped entirely. Folders with no authored index still get the normal
  listing.
